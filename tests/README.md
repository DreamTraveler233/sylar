# 测试目录

本目录包含项目测试，并按类别进行归类：

- `integration/` —— 集成测试，用于端到端地验证系统（如媒体上传流程）。
- `unit/` —— 单元测试，针对各个组件（目前为空，请在此放置 `*.cpp` 文件）。

测试的构建与运行
1. 构建项目：`./autobuild.sh`
2. 运行测试二进制文件：`./bin/tests/test_media`

注意事项：
- 测试目标通过 `tests/CMakeLists.txt` 进行分组。新增测试时请在此添加目标。
- 测试输出文件位于 `bin/tests`，而运行时应用程序二进制文件位于 `bin/app`。
